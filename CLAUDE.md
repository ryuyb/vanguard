# Vanguard Project

A Tauri-based desktop client for Vaultwarden, aiming for 1Password-like usability.

## Critical Rules

- **DO NOT modify this file** unless explicitly requested by the user. Always ask for permission before making changes.
- **Keep responses concise** - be brief and to the point, avoid verbose explanations
- **Use agent teams when needed** - for complex tasks like requirement analysis, code logic analysis, or multi-step workflows

## Tech Stack

- **Backend**: Rust + Tauri
- **Frontend**: React + TypeScript
- **Package Manager**: pnpm
- **UI Components**: shadcn/ui

## Development Guidelines

### Frontend

- Use **feature-based** folder structure
- Separate UI from logic - extract custom hooks
- Use shadcn/ui components whenever possible
- Install missing components: `pnpm dlx shadcn@latest add <component>`
- If installation fails, notify user for manual installation
- **DO NOT modify** files in `src/components/ui/` directly - customize styles at usage sites
- Use `@tauri-apps/plugin-log` for logging (not `console.log`)

### Backend

- Follow **DDD (Domain-Driven Design)** architecture
- Keep methods and files focused and modular - split when necessary
- **DO NOT manually edit** `bindings.ts` - auto-generated by tauri-specta

### Pre-commit Checks

**Rust changes:**
```bash
cargo test
cargo fmt
cargo clippy --all-targets --all-features -- -D warnings
```

**Frontend changes:**
```bash
pnpm run biome:write
pnpm run biome:format
```

Fix all issues before committing.

### Commit Messages

Follow **Conventional Commits** specification:
```
<type>(<scope>): <subject>

<body>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
