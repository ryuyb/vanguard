## ADDED Requirements

### Requirement: 用户可以访问注册页面
系统 SHALL 提供注册页面入口,允许用户从登录页面导航到注册页面。

#### Scenario: 从登录页面导航到注册页面
- **WHEN** 用户在登录页面点击"创建新账户"链接
- **THEN** 系统跳转到注册页面

#### Scenario: 直接访问注册页面
- **WHEN** 用户直接访问 `/register` 路由
- **THEN** 系统显示注册页面

### Requirement: 用户可以输入注册信息
系统 SHALL 提供注册表单,允许用户输入服务器 URL、邮箱和名字。

#### Scenario: 输入有效的注册信息
- **WHEN** 用户输入有效的服务器 URL、邮箱和名字
- **THEN** 系统启用"下一步"按钮

#### Scenario: 输入无效的邮箱格式
- **WHEN** 用户输入无效的邮箱格式
- **THEN** 系统显示邮箱格式错误提示并禁用"下一步"按钮

#### Scenario: 服务器 URL 为空
- **WHEN** 用户未输入服务器 URL
- **THEN** 系统显示必填项提示并禁用"下一步"按钮

### Requirement: 系统检测服务器注册配置
系统 SHALL 调用服务器注册 API 并根据响应状态码判断注册场景。

#### Scenario: 服务器不允许注册
- **WHEN** 用户提交注册信息且服务器返回 400 状态码
- **THEN** 系统显示"注册不被允许或邮箱已存在"错误提示

#### Scenario: 服务器需要邮箱验证
- **WHEN** 用户提交注册信息且服务器返回 204 状态码
- **THEN** 系统显示"验证邮件已发送"提示,引导用户检查邮箱

#### Scenario: 服务器支持直接注册
- **WHEN** 用户提交注册信息且服务器返回 200 状态码和 JWT token
- **THEN** 系统显示密码设置表单

### Requirement: 用户可以设置主密码
当服务器支持直接注册时,系统 SHALL 提供密码设置表单,允许用户设置主密码。

#### Scenario: 输入有效的主密码
- **WHEN** 用户输入主密码和确认密码且两者一致
- **THEN** 系统启用"完成注册"按钮

#### Scenario: 密码不一致
- **WHEN** 用户输入的主密码和确认密码不一致
- **THEN** 系统显示"密码不一致"错误提示并禁用"完成注册"按钮

#### Scenario: 密码强度不足
- **WHEN** 用户输入的密码强度低于最低要求
- **THEN** 系统显示密码强度警告

### Requirement: 系统检查密码泄露
系统 SHALL 使用 Have I Been Pwned API 检查用户密码是否在已知泄露数据库中。

#### Scenario: 密码未泄露
- **WHEN** 用户输入的密码未在泄露数据库中
- **THEN** 系统允许继续注册

#### Scenario: 密码已泄露
- **WHEN** 用户输入的密码在泄露数据库中
- **THEN** 系统显示警告提示但仍允许用户继续

### Requirement: 系统完成注册并自动登录
当用户完成密码设置后,系统 SHALL 执行密码学处理、调用注册完成 API 并自动登录。

#### Scenario: 注册成功并自动登录
- **WHEN** 用户点击"完成注册"且所有验证通过
- **THEN** 系统执行密码学处理、调用注册 API、自动登录并跳转到 Vault 页面

#### Scenario: 注册失败
- **WHEN** 注册 API 调用失败
- **THEN** 系统显示错误提示并保持在注册页面

### Requirement: 用户可以返回登录页面
系统 SHALL 在注册流程的任何阶段提供返回登录页面的选项。

#### Scenario: 从注册表单返回登录
- **WHEN** 用户在注册表单页面点击"返回登录"
- **THEN** 系统跳转到登录页面

#### Scenario: 从密码设置表单返回登录
- **WHEN** 用户在密码设置表单页面点击"返回登录"
- **THEN** 系统跳转到登录页面

### Requirement: 系统显示注册进度
系统 SHALL 在注册过程中显示加载状态和进度提示。

#### Scenario: 提交注册信息时显示加载状态
- **WHEN** 用户点击"下一步"提交注册信息
- **THEN** 系统显示加载指示器并禁用表单

#### Scenario: 完成注册时显示进度
- **WHEN** 用户点击"完成注册"
- **THEN** 系统显示"正在创建账户..."、"正在登录..."等进度提示
